# Lessons

- When the user corrects business-specific solar assumptions, treat that as higher priority than generic market defaults and update all shared calculator constants from the source logic first.
- Keep the home-page calculator summary lightweight; detailed finance fields like EMI, payback, or post-subsidy cost should only appear when the user explicitly wants an advanced calculator view.
- When the user wants Hindi-first presentation, convert the visible hero CTAs, major section headlines, and lead-capture prompts into real Hindi script instead of leaving them in Latin Hinglish.
- If business positioning changes from a city-first pitch to a wider district coverage pitch, update the website copy, blog content, metadata, and WhatsApp prefills together so the brand story stays consistent across the whole funnel.
