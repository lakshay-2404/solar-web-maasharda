# Maa Sharda Web App

Next.js web app for Maa Sharda Distributors with:

- solar lead capture
- calculator-assisted enquiry flow
- Google Sheets as the default low-cost backend
- optional webhook mirroring for cheaper/free automation tools
- installable web app shell
- push notification support
- queued lead retry when connectivity is flaky

## Local Development

Run the app:

```bash
npm run dev
```

Build the app:

```bash
npm run build
```

Lint the app:

```bash
npm run lint
```

- `GOOGLE_SHEETS_SPREADSHEET_ID`
- `GOOGLE_SHEETS_LEADS_SHEET`
- `GOOGLE_SHEETS_PUSH_SHEET`

## Google Sheets Schema

The app can create the tabs automatically if the service account has editor access to the spreadsheet. If you want to create them manually first, use these exact tab names and header rows.

### Sheet 1: `Leads`

Default tab name:

- `Leads`

Override with:

- `GOOGLE_SHEETS_LEADS_SHEET`

Header row:

| Column | Header | Notes |
| --- | --- | --- |
| A | `Timestamp` | `en-IN` formatted date/time written by the app |
| B | `Name` | lead name |
| C | `Phone` | normalized phone number, usually `+91XXXXXXXXXX` |
| D | `Source` | page/source tag such as landing page or calculator flow |
| E | `Monthly Bill` | numeric/string bill value if shared |
| F | `System Size` | chosen or recommended size, for example `3 kW` |
| G | `WhatsApp` | WhatsApp link field generated by the app |
| H | `Status` | default written as `New` |

Expected first row:

```text
Timestamp | Name | Phone | Source | Monthly Bill | System Size | WhatsApp | Status
```

### Sheet 2: `PushSubscriptions`

Default tab name:

- `PushSubscriptions`

Override with:

- `GOOGLE_SHEETS_PUSH_SHEET`

Header row:

| Column | Header | Notes |
| --- | --- | --- |
| A | `Endpoint` | browser push subscription endpoint |
| B | `P256DH` | push public encryption key |
| C | `Auth` | push auth key |
| D | `Phone` | optional phone number mapped to the subscription |
| E | `Locale` | browser/user locale |
| F | `Source` | where subscription came from |
| G | `Consent Context` | why/where the user opted in |
| H | `User Agent` | browser/device info |
| I | `Active` | `TRUE` or `FALSE` |
| J | `Created At` | ISO timestamp |
| K | `Updated At` | ISO timestamp |

Expected first row:

```text
Endpoint | P256DH | Auth | Phone | Locale | Source | Consent Context | User Agent | Active | Created At | Updated At
```

## Notes For Sheets Setup

- Share the spreadsheet with the Google service account email from `GOOGLE_SHEETS_CLIENT_EMAIL`.
- Give it editor access.
- The app uses the spreadsheet id from `GOOGLE_SHEETS_SPREADSHEET_ID`.
- Sheets is the readable/default store for lead records and push subscription lookup.
- If you also configure webhook URLs, the app can mirror writes to external automation tools without changing frontend behavior.

## Provider Behavior

### Leads

- App endpoint: `POST /api/leads`
- Default provider: Google Sheets
- Optional mirror: webhook

### Push subscriptions

- Subscribe endpoint: `POST /api/push/subscribe`
- Unsubscribe endpoint: `POST /api/push/unsubscribe`
- Default provider: Google Sheets
- Optional mirror: webhook

### Protected send endpoints

- `POST /api/push/send-announcement`
- `POST /api/push/send-status`

Protect these endpoints with `AUTOMATION_API_SECRET` using either:

- `Authorization: Bearer <secret>`
- `x-automation-secret: <secret>`
